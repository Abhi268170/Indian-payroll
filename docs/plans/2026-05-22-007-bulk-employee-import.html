Bulk Employee Import

Plan 007 2026-05-22 Branch: feat/bulk-employee-import Status: Ready to build

What

A way to add many employees at once by uploading a single CSV file — instead of entering each employee one by one through the add wizard.

The user uploads a file. The system checks it immediately and shows exactly which rows have problems. Once all problems are fixed, the user clicks one button to import everyone.

Why

New customer onboarding

A company switching to this system may have 50, 200, or 500 existing employees. Adding them one by one through the wizard would take hours. Bulk import reduces this to minutes.

Accuracy

Payroll teams already have employee data in spreadsheets. Uploading a CSV from that spreadsheet avoids re-typing errors.

Speed

HR teams need to be live on payroll by month-end. A manual wizard is a bottleneck. Bulk import unblocks day-one setup.

Key Decisions

One file or many?
One flat CSV with all data — basic info, work details, bank, statutory flags, and salary. No separate templates per module. The user fills in what they have; optional fields can be left blank.
All errors up front or partial import?
All errors shown up front. The Import button stays disabled until every row is clean. This prevents partial data — half-imported employees cause payroll run problems.
What if an employee already exists?
A checkbox on the upload screen: "Update existing employees". Unchecked means skip duplicates silently. Checked means overwrite. Default is unchecked (safe).
What if a Department or Location doesn't exist yet?
Row marked as an error with the message "Department 'Marketing' not found — create it in Settings first." The user creates the reference, then re-uploads. This avoids orphaned data.
How fast is the import?
Up to 500 rows: runs immediately, result shown on screen. Over 500 rows: runs in the background (Hangfire job), employees appear in the list as they are created. Threshold can be tuned.

The CSV Template

One file with 36 columns. Red columns are required — leaving them blank will flag the row as an error. Everything else is optional.

Identity and Work

EmployeeNumber FirstName MiddleName LastName Gender DateOfJoining DateOfBirth WorkEmail PersonalEmail MobileNumber FathersName Department Designation WorkLocation EmploymentType

Address

AddressLine1 AddressLine2 City State PinCode

Sensitive (stored encrypted)

PAN Aadhaar

Bank and Payment

PaymentMode BankAccountHolderName BankName BankAccountNumber IFSCCode BankAccountType

Statutory

EpfEnabled EsiEnabled PtEnabled LwfEnabled UAN ESICNumber

Salary

AnnualCTC SalaryStructureTemplate

Department, Designation, and WorkLocation are matched by name (case-insensitive). They must already exist in Settings before importing. AnnualCTC is optional — if left blank the employee is created without a salary assignment, which can be done later.

required leave blank = row error optional leave blank = field skipped encrypted stored encrypted at rest

What Gets Built

Backend — New files

new
Application/Services/IEmployeeImportParser.cs
Interface that takes a CSV stream and returns a list of parsed rows. Kept in Application so it can be used by command handlers without depending on Infrastructure.
new
Application/Commands/Employees/ValidateEmployeeImportCommand.cs
Dry-run handler. Parses the file, checks every row against the rules above, and returns a report. Does not write anything to the database.
new
Application/Commands/Employees/CommitEmployeeImportCommand.cs
Re-validates then writes all employees in one database transaction. Encrypts PAN, Aadhaar, account number, and IFSC before saving. Single SaveChanges at the end — no per-row commits.
new
Infrastructure/Services/EmployeeImportParser.cs
CsvHelper-based CSV parser. Validates that headers match the template exactly. Returns parsed rows as plain records for the command handlers to use.
new
web/public/templates/employee-import.csv
Downloadable template file with all 36 column headers and one example row. Linked from the import page.
new
web/src/pages/employees/ImportEmployeesPage.tsx
The import UI. Handles drag-drop, auto-validation on file drop, error table display, and the import action. Manages state internally: idle, validating, ready, importing, done.
new
tests/.../ValidateEmployeeImportCommandTests.cs
Tests for missing refs, bad PAN, duplicate email, missing required fields, and the all-clear case.
new
tests/.../CommitEmployeeImportCommandTests.cs
Tests for happy path, overwrite=true vs skip, and CTC optional behaviour.
